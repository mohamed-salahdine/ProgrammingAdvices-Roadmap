# 16 - OOP As It Should Be In C#

**Completed:** March 2026

## Course Overview
This course (from Dr. Mohammed Abu-Hadhoud's ProgrammingAdvices roadmap) focuses on applying **true object-oriented programming** in C# — not just syntax, but proper design principles, clean architecture, and professional patterns.

It builds directly on C# Level 1 (Course 14) and emphasizes understanding over memorization, preparing you for real-world .NET development.

## Key Learnings & Takeaways
- Deep understanding of the four OOP pillars in C# context:  
  - **Encapsulation** (access modifiers, properties vs fields, data hiding)  
  - **Abstraction** (abstract classes, interfaces, hiding complexity)  
  - **Inheritance** (proper use, when to avoid deep hierarchies)  
  - **Polymorphism** (method overriding, interfaces, runtime behavior)  
- Best practices for classes, objects, constructors, destructors, static members & static classes  
- Real usage of properties, indexers, and operator overloading  
- SOLID principles applied in C# (especially Single Responsibility, Open-Closed, Dependency Inversion)  
- Composition over inheritance mindset  
- Clean code & maintainable design patterns for scalable systems

## Challenges & Insights
- Shifting from "just writing classes" to thinking in terms of responsibilities, dependencies, and future maintainability  
- Understanding when inheritance is appropriate vs. when interfaces + composition are better  
- Grasping dependency inversion and how it leads to testable, flexible code
